# Person 5: Documentation & Presentation Lead - 5-Day Plan

Your mission is to start on the final deliverables *immediately*. By the time the code is finished, your documents should be 80% complete, just waiting for the final screenshots and results.

---

### **Day 1: Structure & Skeletons**

*   **Task 1: Create the Presentation Deck:** Create a new PowerPoint/Google Slides file. Build the skeleton: Title slide, Agenda, Introduction (The BYOVD Problem), Project Goals, System Architecture (placeholder), Implementation (placeholder), Live Demo (placeholder), Results (placeholder), Conclusion.
*   **Task 2: Outline the Final Report:** Create a new Word/Google Docs file. Set up the main sections: Abstract, 1.0 Introduction, 2.0 Background, 3.0 System Design, 4.0 Implementation, 5.0 Results, 6.0 Conclusion.
*   **Task 3: Storyboard the Video Demo:** In a simple text file, outline the video shot-for-shot:
    1.  Title Screen.
    2.  Show running the plugin on a *benign* driver (e.g., `kbdclass.sys`).
    3.  Show the resulting low-risk output.
    4.  Show running the plugin on a *vulnerable* driver (e.g., `gdrv.sys`).
    5.  Show the high-risk output, highlighting the score and reasons.
    6.  Conclusion screen.

### **Day 2: Populate with Existing Information**

*   **Task 1: Write Report Introduction & Background:** Use `PROJECT_SUMMARY.txt` and `START_HERE.md` to write the first two sections of the report. Define key concepts like IOCTLs, kernel drivers, and Volatility3.
*   **Task 2: Populate Presentation's Opening:** Transfer the key points from the report's intro/background into the presentation slides. Add speaker notes.
*   **Task 3: Draft Poster Content:** Create a new document for the poster. Draft the key text sections: Abstract, Introduction, and a placeholder for Methodology and Results.

### **Day 3: Draft Technical Sections with Mock-ups**

*   **Task 1: Sync with Technical Team:** Meet with Persons 2, 3, and 4. Get the list of target APIs, the planned scoring model, and the approach for DKOM research.
*   **Task 2: Draft Report's "Implementation" Section:** Based on the sync, write a draft of the implementation section. Describe the planned approach for each phase of the project.
*   **Task 3: Create Mock-up "Results" Slides:** In the presentation, create slides with mock-ups of the final results. Insert fake-but-realistic console output screenshots. This allows you to perfect the layout and explanatory text before the real results are available.

### **Day 4: Refine and Review**

*   **Task 1: Draft Report's "System Design" Section:** Get the architecture diagrams from `ARCHITECTURE.md` and insert them into the report. Write text that explains the data flow and components.
*   **Task 2: Peer Review:** Ask one of your teammates to spend 20 minutes reviewing the presentation slides and the report's introduction. Check for clarity and accuracy.
*   **Task 3: Refine Video Storyboard:** Add narration/script text to your video storyboard. What will the voiceover say during each scene?

### **Day 5: Prepare for Final Results**

*   **Task 1: Check on MVP Progress:** Sync with Person 1. Ask to see the current output of the plugin.
*   **Task 2: Prepare for Screenshots/Screen Recording:** Get your screen-capturing tools ready. Plan the exact commands you will run for the video demo so the recording process is smooth.
*   **Task 3: Draft Conclusion:** Write the draft of the Conclusion section for both the report and the presentation. It should summarize the project's achievements and reiterate the importance of this type of tool. You can also pull from the "Future Work" ideas generated by the technical team.
