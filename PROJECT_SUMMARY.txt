KEY CONCEPTS
============

What is iKARMA?
├─→ Analyzes memory dumps to find dangerous drivers
├─→ Detects BYOVD (Bring Your Own Vulnerable Driver) attacks
├─→ Infers capabilities without original driver binaries
└─→ Provides explainable forensic evidence

How does it work?
├─→ Extract IOCTL handlers from memory (Phase 1)
├─→ Disassemble and analyze code (Phase 1)
├─→ Detect dangerous API patterns (Phase 2)
├─→ Score risk with confidence (Phase 2)
└─→ Detect memory tampering (Phase 3)

Why is it unique?
├─→ Works from memory only (no binaries needed)
├─→ Capability-based (not signature-based)
├─→ Explainable results ("because" tags)
└─→ Detects anti-forensic techniques
